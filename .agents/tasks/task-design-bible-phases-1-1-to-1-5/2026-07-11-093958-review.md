# Design system foundation: tokens, glass hierarchy, component scaffolding, and accessibility baseline

This change introduces the Product Design Bible's foundational layers across five commits: a Lucide React icon library install, a full CSS design token system (colors, typography, radii, shadows), a 5-level glassmorphism hierarchy with motion timing utilities, four new components (Footer, Toast, CommandPalette, refactored Header), and an accessibility/engineering pass (skip-to-content, focus-visible, lint cleanup). The approach layers CSS custom properties and utility classes in `globals.css`, maps them through Tailwind CSS 4's `@theme inline`, and builds components on top. Build and lint pass with zero errors/warnings.

**Watch for:** skip-to-content link targets a `<div>` rather than a landmark element (confirmed); the `@theme inline` block duplicates literal values instead of referencing `:root` vars for shadows/radii/typography, creating a maintenance drift risk (confirmed); CommandPalette reconstructs `quickActions` on every render including router closures (likely performance concern at scale).

## High-level view

The token system establishes a complete design vocabulary — brand colors, semantic colors, typography scale, radii, and 5-level shadows — with light/dark variants that swap via the existing `html.dark` class strategy. The `@theme inline` block hard-codes literal values rather than referencing the `:root` custom properties for font sizes, radii, and shadows, which means updates must be made in two places.

The glassmorphism system adds five named tiers (`glass-subtle` through `glass-ultra`) alongside the pre-existing `glass`, `glass-elevated`, etc. classes. Backward compatibility is preserved by keeping old class names intact rather than replacing them, but this creates a dual naming scheme with no mapping between the two. Motion timing tokens and scroll-triggered entrance animations use `animation-timeline: view()` with a `@supports` fallback that shows elements without animation in unsupported browsers.

The component layer adds a Footer (Server Component), a Toast notification system (context + renderer), and a CommandPalette (Cmd+K modal). The Header replaces inline SVGs with Lucide icons. New components follow the 5-level glass hierarchy; pre-existing components still use legacy glass class names.

The accessibility pass adds a skip-to-content link, global `*:focus-visible` ring, and enhances `prefers-reduced-motion` to universally disable animations. The skip link target is a generic `<div>` wrapper rather than a `<main>` landmark, reducing its effectiveness for assistive technology users.

## Skip-to-content targets a non-landmark div

The skip link in `layout.tsx` points to `#main-content`, which is applied to:

```tsx
<div id="main-content" className="relative z-10 flex min-h-full flex-col">
```

This wrapper contains `<OfflineDetector>`, `<RecentlyPlayedTracker>`, page children, `<MiniPlayer>`, `<SettingsFab>`, `<KeyboardShortcuts>`, and `<CommandPalette>`. A screen reader user activating "Skip to content" lands inside a generic container that includes chrome elements rather than the page's primary content. The actual `<main>` element lives inside each page component (e.g., `app/page.tsx`), so the skip link overshoots.

Fix: wrap `{children}` in a `<main id="main-content">` in the layout itself, and remove the per-page `<main>` tags. Alternatively, move `id="main-content"` to the `<main>` inside each page template.

## Token duplication between :root and @theme inline

The `@theme inline` block defines shadow, radius, and font-size values as literal strings identical to those in `:root`. The FEAT-002 findings note explains this is intentional — Tailwind CSS 4's `@theme inline` cannot use `var()` references without self-referencing issues. The consequence is that any design token change must be made in two locations with no compile-time guard against drift.

## CommandPalette re-creates action closures on every render

```tsx
const quickActions: QuickAction[] = [
  { id: "home", label: "Go to Home", icon: Home, action: () => { router.push("/"); setIsOpen(false); } },
  // ... 5 more
];
```

This array is constructed inside the component body without memoization. Each render creates 6 new function objects. The downstream `filteredActions` filter also recomputes on every keystroke. As the command list grows (search results, history items, plugin commands), this pattern will cause unnecessary re-renders in child elements that depend on referential equality.

## Dual glass naming without mapping

The CSS now contains two naming schemes:

- Legacy (13 classes): `glass`, `glass-elevated`, `glass-search`, `glass-dropdown`, `glass-player`, `glass-card`, `glass-tooltip`, `glass-sidebar`, `glass-dialog`, `glass-settings`, `glass-recommendation`, `glass-stats`, `glass-fab`
- Design Bible 5-level: `glass-subtle`, `glass-light`, `glass-medium`, `glass-heavy`, `glass-ultra`

New components use the 5-level system (CommandPalette = `glass-heavy`, Toast = `glass-light`, Footer = `glass-subtle`). Pre-existing components still use legacy names (Header = `glass-elevated`). Without a mapping comment or deprecation notice, contributors will arbitrarily choose between the two schemes.

## Global focus-visible may conflict with component-specific focus styles

The universal rule applies `outline: 2px solid var(--primary)` to every focusable element. Components that need a different focus treatment (inset ring, box-shadow, or suppressed outline like the skip link's `focus:outline-none`) must explicitly override. The skip link suppresses the outline but relies on becoming visible as its focus indicator — acceptable for a skip link, but the pattern means any custom focus styling requires fighting the global rule's specificity.
