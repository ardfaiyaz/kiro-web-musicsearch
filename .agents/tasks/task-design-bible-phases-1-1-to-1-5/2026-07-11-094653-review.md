## Skip link targets a div instead of a main landmark

The fix commit moved `id="main-content"` to wrap only `{children}`, which correctly excludes chrome elements (MiniPlayer, SettingsFab, etc.). However the element is:

```tsx
<div id="main-content" tabIndex={-1}>
  {children}
</div>
```

Screen readers that expose a landmark list (VoiceOver's rotor, NVDA's landmark navigation) won't surface this as a content landmark. Each page component (`app/page.tsx`, `app/discover/page.tsx`, etc.) still renders its own `<main>` element inside this wrapper, creating a `<div>` → `<main>` nesting where the skip link lands on the outer non-landmark container.

Replacing the `<div>` with `<main>` and removing the per-page `<main>` tags would fix landmark navigation and eliminate the nested-main pattern.

## Unconditional transition override on animate classes

```css
[class*="animate-"] {
  transition: none;
}
```

This selector matches every element with "animate-" anywhere in its class attribute — including Tailwind's built-in `animate-pulse`, `animate-spin`, `animate-bounce`, and any component using custom animation classes. It unconditionally strips CSS transitions from these elements, which breaks hover/focus transitions on animated elements even when the user has not requested reduced motion.

The `@media (prefers-reduced-motion: reduce)` block that follows already sets `transition-duration: 0.01ms !important` on all elements, making this rule redundant for the reduced-motion case and harmful for the normal case.
