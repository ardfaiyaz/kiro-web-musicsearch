# Cursor-pointer added to all interactive elements

This change adds the `cursor-pointer` Tailwind utility class to every clickable element across 11 component and page files: buttons, links (`<Link>` and `<a>`), a `<select>` dropdown, and clickable card wrappers. The motivation is that Tailwind CSS 4's preflight resets the default cursor on `<button>` elements to `default` rather than `pointer`. The approach is a straightforward class addition â€” no behavioral logic changes, no new abstractions.

**Watch for:** Redundant `cursor-pointer` on elements that already receive it from the browser (links with `href`) [possible].

## High-level view

For buttons and the select dropdown, this is a necessary fix in Tailwind v4 where preflight sets `cursor: default` on button elements. For `<a>` and `<Link>` elements that already have an `href`, browsers already apply `cursor: pointer`, so the class is redundant but harmless â€” it makes the intent explicit and guards against future CSS resets.

## Redundant cursor-pointer on links

The `<Link>` component from Next.js renders an `<a>` with `href`, so `TrackCard`, `AlbumCard`, `ArtistCard`, and all navigation links in `page.tsx`, `favorites/page.tsx`, and `track/[id]/page.tsx` already had the pointer cursor before this change. The elements where `cursor-pointer` is genuinely needed are the `<button>` elements (SearchFilters filter buttons, Clear Filters, SearchBar submit, FavoriteButton, AudioPlayer play/pause and mute) and the `<select>` in SearchFilters â€” these are the ones affected by Tailwind v4's preflight reset.
