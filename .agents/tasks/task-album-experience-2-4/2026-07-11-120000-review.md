# Cinematic album page with dynamic color extraction and modular component architecture

A complete rewrite of the album page from a single Server Component with an inline table-based tracklist into a 20-component modular architecture. The page now fetches enrichment data in parallel (Spotify, artist discography, similar artists) via `Promise.allSettled`, delegates interactive sections (hero, sticky bar, tracklist, description) to a single `AlbumPageClient` boundary, and renders metadata sections (credits, details, similar albums, videos, timeline, external links) as Server Components. Each album derives a unique color palette from its artwork via canvas-based extraction, applied to the background gradient, ambient glow, and UI accents.

**Watch for:** `AlbumDescription` uses `dangerouslySetInnerHTML` without sanitization — currently passed `null`, but the architecture invites XSS if description sources are connected later (confirmed). `AlbumStats.popularity` is always `undefined` due to a no-op ternary (`spotify ? undefined : undefined`), so the stat never renders (confirmed). `AlbumCredits` displays `spotify.artistName` labeled "Distributed by", but this field is the album artist name from Spotify search, not the record label (confirmed).

## High-level view

The page architecture cleanly separates server-fetched data from client-side interactivity. A single `AlbumPageClient` component wraps the hero, sticky bar, tracklist, and description — the minimal surface that needs `"use client"`. `Promise.allSettled` in the page component means any single API failure (Spotify, Last.fm) gracefully hides the corresponding section rather than killing the entire page.

The dynamic color system extracts palettes client-side via canvas. Because extraction is asynchronous, the hero background renders with static fallback colors initially, then shifts to extracted colors once the artwork image loads. The `opacity-60` gradient layer and dark overlays reduce the visual impact, but on slow connections the color shift is perceptible — particularly the bloom element which pulses in one color then transitions mid-animation.

The diff also fixes issues from the previous review pass: `ArtistHero` parallax now writes directly to a ref (no per-frame re-renders), `ArtistNav` consolidates to a single scroll listener with ref-based batching, `AIDiscoveryPanel` routes through `/api/recommendations` with `AbortController` cleanup, and `PersonalizedSection` now reverts mood and shows an error banner on failure.

1. **XSS vector in AlbumDescription** — `dangerouslySetInnerHTML` renders unsanitized HTML. Currently safe only because `null` is always passed. Add DOMPurify or strip HTML before rendering when a real source is connected.
2. **Popularity stat never renders** — `popularity={spotify ? undefined : undefined}` is a no-op; the Spotify album type lacks a `popularity` field, but the code should either remove the stat or source it from another API.
3. **Credits shows artist name as distributor** — `spotify.artistName` is displayed with "Distributed by" label. `SpotifyAlbum` has no `label` field, so this should either be removed or the interface extended to include label data.

5. **Color extraction CORS dependency** — `img.crossOrigin = "anonymous"` on iTunes CDN images works today but depends on permissive CORS headers. Fallback is graceful, but the hero flashes from fallback to extracted colors on late resolution.
6. **AnimatedCounter fires immediately on mount** — The counter animates regardless of viewport visibility. Acceptable for above-the-fold hero stats but would waste motion for any future use below the fold.
7. **MusicVideos fetch not cached across navigations** — Each visit re-fetches YouTube results. For a rarely-changing resource, sessionStorage or SWR caching would reduce API pressure.

## XSS surface in AlbumDescription

`AlbumDescription` renders its `description` prop via `dangerouslySetInnerHTML`:

```tsx
<p
  className="text-sm leading-relaxed text-muted"
  dangerouslySetInnerHTML={{ __html: description }}
/>
```

The `AlbumPageClient` currently passes `description={null}`, which means the empty-state branch always renders. But the component is architecturally ready to accept Spotify editorial notes or Last.fm album info — both of which return HTML-like content from third-party APIs. The moment a real data source connects, this becomes an XSS vector unless the input is sanitized first. Either add `dompurify` as a dependency and sanitize, or switch to rendering plain text with `whitespace-pre-line` styling.

## Spotify popularity dead code

```tsx
popularity={spotify ? undefined : undefined}
```

Both branches of the ternary return `undefined`. The intent appears to be `popularity={spotify?.popularity}`, but `SpotifyAlbum` doesn't have a `popularity` field (only `SpotifyTrack` does). The stat card for "Popularity" therefore never renders. Either remove it from `AlbumStats` or source popularity from the Spotify artist endpoint where it is available.

## Credits mislabeling artistName as distributor

`AlbumCredits` renders a card when `spotify` is present:

```tsx
<p className="truncate text-sm font-semibold text-foreground">
  {spotify.artistName}
</p>
<p className="text-xs text-muted">Distributed by</p>
```

`SpotifyAlbum.artistName` is the performing artist ("Taylor Swift"), not the distribution label ("Republic Records"). The `SpotifyAlbum` interface doesn't include a `label` field. This card either needs to be removed, or the Spotify API response needs to be expanded to include the `label` property that the full album endpoint does provide.

## Color palette flash on hydration

`DynamicBackground` renders static fallback gradient colors during initial paint, then asynchronously extracts colors from the artwork image via canvas. On slow connections where the artwork image loads late, the bloom `animate-pulse` div uses the fallback color until extraction completes, which can cause the bloom to change color mid-animation. The `opacity-60` gradient layer and dark overlays reduce the visual impact to a subtle shift rather than a jarring change, but it remains perceptible on slower networks.
